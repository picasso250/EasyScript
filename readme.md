

# Axiom (易通) 脚本语言：设计与哲学

## 简介

**Axiom (易通)** 是一种为追求极致**简洁**、**易学**和**结构清晰**而设计的脚本语言。它是一门**图灵完备**的语言，具备现代编程所需的所有核心功能（变量、赋值、复合数据、控制流），但通过严格限制规则，将学习和实现的复杂度降至最低。

**Axiom** 的目标是成为初学者的完美**基石 (Axiom)** 语言，让学习者能专注于算法逻辑，而非记忆复杂的语法和优先级规则。

## 核心设计哲学

Axiom 的设计围绕两大核心原则：

### 1. 极致的 BNF 简洁性

我们通过牺牲传统语言中复杂且不常用的优先级层级，换取一个极其简洁的 BNF 结构。

*   **表达式层级：仅两层优先级**
    1.  **高优先级（Term）：** 乘除、取模、所有位运算、所有移位运算。
    2.  **低优先级（Expression）：** 加减、所有关系运算、所有相等性运算、所有逻辑运算。
*   **强制使用括号：** 所有不同类别操作符（例如算术与逻辑）之间的优先级差异，必须使用圆括号 `()` 来明确，**杜绝了优先级歧义**，极大地提升了初学者的可读性。
*   **最小化非终结符：** 表达式结构仅由 $\langle \text{Primary} \rangle \to \langle \text{Factor} \rangle \to \langle \text{Term} \rangle \to \langle \text{Expression} \rangle$ 四个非终结符构成。

### 2. 结构化和初学友好

Axiom 的控制流借鉴了现代语言的最佳实践，旨在消除常见的编程陷阱。

*   **Go-Style 控制流：** `if` 和 `for` 语句的条件表达式**无需使用圆括号**。
*   **强制代码块：** `if`, `else`, `for` 的语句体**必须**是代码块 `{}` ($\langle \text{Block} \rangle$)。这彻底解决了“悬挂 `else`”问题，并强制要求编写结构清晰、可维护的代码。
*   **简洁的循环：** `for` 语句集成了 Go 语言的两种简洁形式（`for condition { ... }` 和 `for init; condition; post { ... }`）。

## 语言核心特性速览

### 数据结构

| 特性 | 实现方式 |
| :--- | :--- |
| **变量 (Identifier)** | 支持标识符的赋值和读取。 |
| **基础值** | 支持 **Number**、**true** 和 **false**。 |
| **列表 (List)** | 支持使用字面量 `[1, a+b, true]` 构造。 |
| **高优先级索引** | 列表索引操作 `[ ]` 具有**最高的优先级**，可以对任何表达式或列表字面量进行即时访问，例如 `[1,2,3][1]`。 |

### 运算符

Axiom 包含了 C 语言中所有必要的二元运算符，并将其统一分配到两个优先级层级：

*   **高优先级 (`Term`)：** `*`, `/`, `%`, `<<`, `>>`, `&`, `|`, `^`
*   **低优先级 (`Expression`)：** `+`, `-`, `<`, `<=`, `>`, `>=`, `==`, `!=`, `&&`, `||`

### 控制流

| 语句 | 示例 (Go-Style) |
| :--- | :--- |
| **赋值** | `myVar = 10 * (a + b);` |
| **条件分支** | `if a > 10 { result = 1; } else { result = 0; }` |
| **循环 (While-Style)** | `for i < 10 { i = i + 1; }` |
| **循环 (C-Style)** | `for i=0; i<10; i=i+1 { sum = sum + i; }` |

## 总结

**Axiom (易通)** 证明了**极简**与**功能强大**可以完美并存。通过精心设计的 BNF 结构，我们创造了一个易于实现、易于学习、且完全具备执行任何算法能力的脚本语言。它是学习编程语言设计和算法逻辑的理想选择。
